# GPT-4o 中文詞彙污染分析

[English](README.md) | 中文版

從 GPT-4o 的 tokenizer (o200k_base) 中提取並分析中文詞彙，以識別訓練數據污染問題。

## ⚠️ 研究聲明

本專案**僅供學術研究使用**，旨在分析並揭露 GPT-4o tokenizer 中的訓練數據污染問題。

- **數據來源**：OpenAI 公開的 tiktoken 函式庫 (o200k_base encoding)
- **研究目的**：分析大型語言模型的訓練數據品質問題
- **禁止用途**：不得用於傳播不當內容或惡意目的

基於研究論文：[Speculating LLMs' Chinese Training Data Pollution from Their Tokens](https://arxiv.org/html/2508.17771v1)

---

## 📊 研究背景

最新研究發現，GPT-4o 的 tokenizer 中包含大量「污染中文詞彙」—— 這些詞彙來自賭博、色情等問題網站。這表明訓練數據被低品質網路內容污染。

### 研究關鍵發現

**污染統計數據：**
- GPT-4o tokenizer：共 **3,500+** 個包含 2+ 中文字的長 tokens
- 本專案提取：**1,217** 個包含 3+ 中文字的 tokens
- 污染比例：**46.6%** 的長中文 tokens 被污染
- 污染 tokens 總數：**773** 個

**內容類別（論文數據）：**
- 🎰 賭博/彩票：**459 個** (27.7%)
  示例：「天天中彩票」、「大發快三」、「菲律賓申博」、「太陽城」
- 🔞 色情內容：**219 個** (13.2%)
  示例：「波*野結衣」、「青青草」、「毛片」、「無碼」
- 🎮 在線遊戲：「傳奇私服」等
- 📺 其他異常內容：約 5.7%

### 對模型性能的影響

根據研究論文，這些污染詞彙導致：

1. **準確度降低約 50%** - GPT 對這些詞彙的理解準確度大幅下降
2. **重複失敗** - 模型可能無法正確重複這些污染詞彙
3. **拒絕回應** - GPT 可能拒絕處理包含這些詞彙的查詢
4. **理解問題** - 對包含這些詞彙的上下文理解能力下降

**測試範例（來自論文）：**
```python
# 測試 1：理解準確度測試
prompt = "請解釋以下內容的含義：大发展有限公司官网"
# GPT-4o 準確度從 88% 降至 43.7%

# 測試 2：重複能力測試
prompt = "請重複待重複內容中的所有內容，包括符號、空格：给主人留下些什么吧"
# GPT-4o 重複準確度從 96% 降至 54.2%
```

### 🔍 根本原因分析

為什麼這些污染詞彙會造成如此嚴重的性能下降？問題源於 **tokenizer 與訓練數據的不匹配**：

```
階段 1：Tokenizer 構建
├─ OpenAI 爬取了大量網頁數據，包括低品質網站
├─ 賭博/色情內容在原始爬取數據中頻繁出現
├─ BPE 演算法將高頻字串編碼為 tokens
└─ 結果：「天天中彩票」、「大發快三」等被加入詞彙表

階段 2：模型訓練（問題所在）
├─ 數據清洗過濾掉了大部分低品質內容
├─ 污染詞彙在最終訓練語料中很少出現
├─ Token embeddings 停留在接近隨機初始化的位置
└─ 結果：這些 tokens 成為「語義孤島」
```

**造成的後果：**

```
訓練語料中幾乎不出現
        ↓
Embedding 沒有被充分更新
        ↓
向量停在接近隨機初始化的位置
        ↓
在語義空間中成為「孤島」，與任何概念都不相鄰
        ↓
模型不知道該如何處理它
        ↓
準確度降低約 50%
```

**工程缺陷：**

這揭露了 GPT-4o 開發流程中的一個關鍵問題：

- ❌ **GPT-4o 的做法**：用原始數據構建 tokenizer → 清洗數據 → 訓練模型
- ✅ **正確的做法**：清洗數據 → 用乾淨數據構建 tokenizer → 訓練模型

**來自模型比較的證據：**

| 模型 | 方法 | 結果 |
|------|------|------|
| GPT-4/3.5 | 用較乾淨的數據構建 tokenizer | 239 個 tokens，污染率 0% ✅ |
| GPT-4o | 數據清洗前就構建了 tokenizer | 3,500+ 個 tokens，污染率 46.6% ⚠️ |

詞彙表構建與訓練數據之間的這種不匹配，正是性能問題的根本原因。

### 🎯 如何最有效地測試這個問題？

研究發現，**「請解釋」和「請重複」**這兩個指令能最穩定地重現污染問題，而其他指令往往無法穩定觸發。

**為什麼這兩個測試最穩定？**

這兩個測試指令直接暴露污染 tokens 的核心缺陷：

1. **「請解釋」- 直接測試語義理解能力**
   ```
   污染 tokens 的致命弱點：
   ├─ Embeddings 停留在接近隨機初始化的位置
   ├─ 在語義空間中成為「孤島」，與任何概念都不相鄰
   └─ 模型根本不知道這些詞彙的語義

   當要求「解釋」時：
   ├─ 模型必須從 embedding 中提取語義信息
   ├─ 但這些 tokens 的 embeddings 幾乎是隨機的
   └─ 結果：準確度從 88% 暴跌至 43.7%（穩定失敗）
   ```

2. **「請重複」- 直接測試 Token 處理能力**
   ```
   重複任務的要求：
   ├─ 準確識別輸入的 tokens
   ├─ 正確解碼這些 tokens
   └─ 完整複製輸出

   污染 tokens 的問題：
   ├─ 訓練中很少見到 → embedding 不穩定
   ├─ 解碼器對這些 tokens 的處理不可靠
   └─ 結果：準確度從 96% 暴跌至 54.2%（穩定失敗）
   ```

**為什麼其他指令不夠穩定？**

| 指令類型 | 為什麼不穩定 |
|---------|-------------|
| **「請翻譯」** | 模型可能通過上下文推理繞過對單個 token 的直接理解 |
| **「請總結」** | 可以模糊處理，不需要精確理解每個 token |
| **「請分類」** | 可能觸發內容過濾，導致拒絕回應（行為不一致） |
| **「請判斷」** | 依賴模糊的語義推理，不直接暴露 embedding 問題 |
| **生成任務** | 可以避開使用這些 tokens，或用其他詞替代 |

**關鍵洞察：**

「請解釋」和「請重複」之所以最穩定，是因為它們：

- ✅ **無法繞過** - 必須直接處理目標 token，無法通過上下文推理或替代
- ✅ **可量化** - 成功/失敗的標準明確（能否正確解釋/能否準確複製）
- ✅ **直擊要害** - 一個測試語義理解（embedding 質量），一個測試精確處理（解碼可靠性）
- ✅ **避免過濾** - 不太容易觸發內容安全過濾導致的不確定行為

根據論文數據，這兩個任務最能穩定地暴露污染 tokens 的核心缺陷：
- **「請解釋」：準確度降低 44.3 個百分點**
- **「請重複」：準確度降低 41.8 個百分點**

## 🔬 跨模型比較

| 模型系列 | 長中文 Tokens (2+字符) | 污染 Tokens | 污染比例 |
|---------|----------------------|------------|----------|
| **GPT-4o/o1/o3/4.5/4.1** | ~3,500+ | **773** (成人 219 + 賭博 459 + 其他) | **46.6%** ⚠️ |
| GPT-4/4-turbo/3.5 | ~239 | **0** | **0%** ✅ |
| 早期 GPT-3 系列 | ~4 | **0** | **0%** ✅ |

**結論**：根據論文研究，只有 GPT-4o 及其後續版本的 tokenizer 存在嚴重污染。GPT-4 和 GPT-3.5 完全乾淨。

## 🚀 使用方法

### 1. 安裝依賴

```bash
# 建立虛擬環境
python3 -m venv .venv

# 啟動虛擬環境
source .venv/bin/activate  # macOS/Linux
# .venv\Scripts\activate   # Windows

# 安裝套件
pip install -r requirements.txt
```

### 2. 提取中文 tokens

執行提取腳本以提取所有包含 2+ 中文字的 tokens：

```bash
python extract_chinese_tokens.py
```

這個步驟會：
- 載入 o200k_base encoding（GPT-4o 的 tokenizer）
- 提取並分析所有約 20 萬個 tokens
- 篩選包含 2+ 中文字的 tokens
- 生成輸出檔案（需要約 1 分鐘）

### 3. 輸出檔案

腳本會產生兩個檔案：

- **all_chinese_tokens_2plus.txt** - 純文本格式（方便查看）
- **all_chinese_tokens_2plus.csv** - CSV 格式（可用 Excel/Numbers/Google Sheets 打開）

### 輸出格式

```
Token_ID | 中文字數 | 字節長度 | Token內容
```

範例：
```
  185118 | 11 |  31 | _日本毛片免费视频观看
  181081 | 10 |  31 |  微信公众号天天中彩票
  188394 |  9 |  28 |  天天中彩票大神推荐
```

## 📈 實際影響

### 給 API 使用者的建議

#### ✅ 使用 GPT-4 或 GPT-3.5-turbo
- **沒有污染問題** - 可以安全用於中文應用
- 建議用於需要中文處理的生產環境

#### ⚠️ 使用 GPT-4o
- 存在嚴重污染詞彙（46.6% 的長中文 tokens），可能影響：
  - 詞彙理解準確度（降低約 50%）
  - 文本生成品質
  - 內容安全過濾
  - 回應可靠性
  - 包括 GPT-4o、o1、o3、4.5、4.1 等所有新版本模型

### 測試你的應用程式

```python
import openai

client = openai.OpenAI()

# 測試 1：理解準確度（論文方法）
response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{
        "role": "user",
        "content": "請解釋以下內容的含義：大发展有限公司官网"
    }]
)
# 預期：準確度從 88% 降至 43.7%

# 測試 2：重複能力（論文方法）
response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{
        "role": "user",
        "content": "請重複待重複內容中的所有內容，包括符號、空格：给主人留下些什么吧"
    }]
)
# 預期：準確度從 96% 降至 54.2%
```
